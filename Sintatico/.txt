Para gerar uma árvore sintática abstrata (AST - Abstract Syntax Tree) a partir do seu código Flex e Bison (Yacc), é necessário modificar o arquivo do parser para que, além de reconhecer a sintaxe, ele também construa uma estrutura de dados em árvore que represente o código-fonte analisado.

Isso é feito através da adição de "ações semânticas" às regras da gramática. Essas ações são blocos de código em C que são executados sempre que uma regra é reconhecida.

Aqui estão os passos detalhados para transformar seu parser em um construtor de AST:

Passo 1: Definir as Estruturas de Dados para os Nós da Árvore (em C)

Primeiro, você precisa definir como será um nó da sua árvore. Como existem diferentes tipos de construções na linguagem (declarações, expressões, comandos, etc.), é comum usar uma struct genérica para o nó e uma union ou um enum para especificar o tipo de nó e seus dados específicos.

Crie um arquivo de cabeçalho (ex: ast.h) para conter essas definições.

Exemplo de ast.h:

code
C
download
content_copy
expand_less
#ifndef AST_H
#define AST_H

// Enum para identificar o tipo de nó
typedef enum {
    NODE_VAR_DECL,
    NODE_FUN_DECL,
    NODE_COMPOUND_STMT,
    NODE_IF_STMT,
    NODE_WHILE_STMT,
    NODE_RETURN_STMT,
    NODE_ASSIGN_EXPR,
    NODE_BINARY_OP,
    NODE_NUM,
    NODE_VAR,
    NODE_FUN_CALL
    // ... outros tipos de nós que você precisar
} NodeType;

// Estrutura genérica para um nó da AST
typedef struct ASTNode {
    NodeType type;
    struct ASTNode *leftChild;
    struct ASTNode *rightChild;
    // Irmão para listas (ex: lista de comandos, lista de parâmetros)
    struct ASTNode *next;
    // Dados específicos do nó
    union {
        int number;
        char* identifier;
        // ... outras informações, como tipo da variável, operador, etc.
    } data;
} ASTNode;

// Funções para criar nós (protótipos)
ASTNode* create_node(NodeType type, ASTNode* left, ASTNode* right);
ASTNode* create_leaf_num(int value);
ASTNode* create_leaf_id(char* name);
// ... outras funções auxiliares

#endif

Você também precisará de um arquivo ast.c para implementar as funções de criação de nós, como create_node, que alocam memória para um novo nó e inicializam seus campos.

Passo 2: Modificar a Seção de Definições do Parser (.y file)

No seu arquivo do parser, você precisa informar ao Bison sobre os tipos de dados que serão associados aos símbolos da gramática (tokens e não-terminais).

Incluir o cabeçalho da AST:

code
Yacc
download
content_copy
expand_less
%{
#include "ast.h"
// ... resto do seu código %{ %}
%}

Definir a união de tipos (%union):
A diretiva %union define todos os tipos de valores que yylval (a variável que armazena o valor semântico de um token) e $$ (o valor semântico de uma regra) podem ter. Para construir uma AST, o tipo mais importante será um ponteiro para um nó da árvore.

code
Yacc
download
content_copy
expand_less
%union {
    int number;
    char* string;
    struct ASTNode *node; // Ponteiro para um nó da AST
}

Associar tipos aos símbolos:
Agora, você precisa dizer ao Bison qual campo da %union usar para cada símbolo.

Tokens: Use %token <campo_da_union> NOME_DO_TOKEN.

Não-terminais: Use %type <campo_da_union> nome_do_nao_terminal.

code
Yacc
download
content_copy
expand_less
// ...
%token <number> NUM
%token <string> ID // Assumindo que você modificará o Flex para alocar e retornar strings
%token IF ELSE INT RETURN VOID WHILE /* ... outros tokens sem valor específico */

// Associando não-terminais que representarão sub-árvores a um ponteiro de nó
%type <node> programa declaracao_lista declaracao var_declaracao fun_declaracao
%type <node> composto_decl statement_lista statement expressao_decl selecao_decl
%type <node> iteracao_decl retorno_decl expressao simples_expressao soma_expressao
%type <node> termo fator var ativacao
// ... e todos os outros não-terminais que devem resultar em um nó na AST
Passo 3: Modificar o Scanner (Flex) para lidar com Valores

Seu scanner precisa colocar o valor apropriado na variável yylval.

Para números, você já está fazendo isso: yylval=atoi(yytext);. Com a %union, você precisará especificar o campo: yylval.number = atoi(yytext);.

Para identificadores (ID), você precisa copiar o yytext (que é temporário) para uma memória alocada dinamicamente e atribuir o ponteiro a yylval.string.

Exemplo de modificação no arquivo .l:

#include "parser.tab.h"

{numero}      { yylval.number = atoi(yytext); return NUM; }
{identificador} { yylval.string = strdup(yytext); return ID; } // Usa strdup para alocar memória

Passo 4: Adicionar as Ações Semânticas às Regras da Gramática

Este é o passo principal. Para cada regra no seu arquivo .y, você adicionará um bloco de código {...} que cria um ou mais nós da AST e os conecta.

$$ representa o valor semântico da regra (o lado esquerdo). Você deve atribuir o nó raiz da sub-árvore criada a $$.

$1, $2, etc., representam os valores semânticos dos componentes da regra (do lado direito).

Exemplo de modificação das regras:

code
Yacc
download
content_copy
expand_less
%%

/* ... */

/* 4. Declaração de variável */
var_declaracao: tipo_especificador ID SEMICOLON
              {
                  // $2 é o valor do token ID (um char*)
                  // Criamos um nó para a declaração de variável
                  // A função create_var_decl_node seria uma função auxiliar sua em ast.c
                  $$ = create_var_decl_node($2);
              }
              | tipo_especificador ID OPENCOL NUM CLOSECOL SEMICOLON
              {
                  // $2 é o ID (char*), $4 é o NUM (int)
                  $$ = create_array_decl_node($2, $4);
              }
              ;

/* ... */

/* 15. Comando de seleção (if/else) */
selecao_decl: IF OPENPAR expressao CLOSEPAR statement %prec IFX
            {
                // $3 é o nó da expressão (condição)
                // $5 é o nó do statement (corpo do if)
                $$ = create_if_node($3, $5, NULL); // NULL para o else
            }
            | IF OPENPAR expressao CLOSEPAR statement ELSE statement
            {
                // $3 é a condição, $5 o corpo do if, $7 o corpo do else
                $$ = create_if_node($3, $5, $7);
            }
            ;

/* 22. Expressão de soma/subtração */
soma_expressao: soma_expressao ADD termo
              {
                  // $1 é o nó da sub-árvore à esquerda
                  // $3 é o nó da sub-árvore à direita
                  // create_binary_op_node é uma função sua para criar um nó de operação
                  $$ = create_binary_op_node(ADD, $1, $3);
              }
              | soma_expressao SUB termo
              {
                  $$ = create_binary_op_node(SUB, $1, $3);
              }
              | termo
              {
                  $$ = $1; // A sub-árvore é simplesmente o nó do termo
              }
              ;

/* 26. Fator */
fator: OPENPAR expressao CLOSEPAR
     {
         $$ = $2; // Repassa o nó da expressão interna
     }
     | var
     {
         $$ = $1;
     }
     | ativacao
     {
         $$ = $1;
     }
     | NUM
     {
         // $1 é o valor do token NUM (um int)
         $$ = create_leaf_num($1); // Cria um nó folha para o número
     }
     ;

%%```

### Passo 5: Armazenar a Raiz da Árvore

A regra inicial (`programa`) será a responsável por finalizar a árvore. Você precisará de uma variável global (por exemplo, `ASTNode *root;`) para armazenar o ponteiro para a raiz da árvore completa.

```yacc
%{
#include <stdio.h>
#include <stdlib.h>
#include "ast.h" // Incluir seu cabeçalho

ASTNode *root = NULL; // Variável global para a raiz da AST

// ...
%}

// ... (union, tokens, types)

%start programa

%%

/* 1. Regra inicial */
programa: declaracao_lista
        {
            root = $1; // Salva a árvore completa na variável global
            printf("Árvore sintática construída com sucesso.\n");
        }
        ;

// ... resto das regras com ações

Após a chamada de yyparse() no seu main, se ela retornar sucesso, a variável root conterá a árvore sintática completa, pronta para ser usada por fases posteriores do compilador, como a análise semântica ou a geração de código.
